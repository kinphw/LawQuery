> 사업 정체성의 핵심 문서. 기능 설계·우선순위·유료화 판단의 기준.
> (Claude·GPT·Gemini 3개 AI 의견 + 운영자 박건영(회계사·금감원 업무경험) 판단 종합, 2026-06)

## 1. 제품 정체성

- **모브랜드(가칭 Kinphw Query) 아래 동등한 전문 검색·연계 서비스들**
  - **LawQuery**: 전자금융거래법·여신전문금융업법·금융위 법령해석·비조치의견서
  - **AccountingQuery**(예정): 회계기준(K-IFRS)·BC·질의회신
  - 장기: TaxQuery, AuditQuery 등
- **본질은 "검색"이 아니라 "전문가가 큐레이션한 연계 정보·해설 플랫폼"**
  - 조문/기준서 단순 검색은 누구나 국가법령정보센터·기준원에서 가능 → 무료여야 함
  - 돈을 내는 이유 = **시간 절약** + **혼자선 만들 수 없는 연계뷰**
- **웹이 본질, 안드로이드 앱(Play Store)은 발견·홍보 채널** (앱은 결제 없는 TWA 뷰어)
- 운영자 경쟁력: **회계사 + 금융감독 업무경험** → 자료 신뢰도·해설이 해자(moat)

## 2. ★ 킬 기능 (Kill Features) — 사업의 핵심

이게 경쟁사가 못 하는 것이자 유료화의 근거. **free=미끼, pro=킬**로 게이팅.

### 법령 (LawQuery)
- **미끼(free)**: 법령 본문 조회
- **★ 킬(pro)**:
  - **5단 연계표** (법→시행령→감독규정→시행세칙→별표) — 금감원도 내부에서 제본해 쓰는 그 표를 디지털로. **최강 킬 기능.**
  - **별표 팝업** 뷰어
  - **법령해석·비조치의견서 검색** — 금융위/금감원 자료를 크롤링·가공. 공식 사이트가 HWP라 검색이 악명 높게 어려운 걸 보기 쉽게. **조회 자체가 큰 가치.**

### 회계 (AccountingQuery, 예정)
- **미끼(free)**: 기준서 본문 조회
- **★ 킬(pro)**:
  - **기준서 본문 ↔ BC(Basis for Conclusions) 2단 연계뷰** — 본문과 결론근거를 함께 보는 것. 회계 실무에서 가장 중요. **경쟁사 k-ifrs.co.kr이 못 하는 차별점.**
  - 질의회신 연계

### 왜 이게 돈이 되는가 (3 AI 공통 결론)
- "검색"엔 돈 못 받지만 **큐레이션·연계정보·업데이트 알림·전문가 해설**엔 돈 받음
- 운영자는 이미 그 킬 기능을 핵심으로 보유 → 운 좋게 "돈 되는 기능"이 본질

## 3. 수익화 로드맵

| 단계 | 시점 | 내용 |
|---|---|---|
| **1. 무료 베타 + 기능 차등** | 지금 | 가입 자동승인. **free=미끼 / pro=킬** 구조를 처음부터 구현. 단 pro는 **관리자 수동 부여**(결제 없음). Play Store 무료 등록 |
| **2. Pro 결제 실험** | 활성 100명 수준 | 웹 PG 정기결제 → `plan=pro` 자동 부여. (GPT/Gemini: "1000명 기다리지 말고 100명에서 실험") |
| **3. 통합 멤버십** | AccountingQuery 출시 후 | Law·Accounting 묶음. 객단가↑ |
| **4. 팀 라이선스** ★ | 검증 후 | 금융사·회계법인·법무팀 대상 월 5~20만 원. **개인 구독보다 큰 수익원** |

### 핵심 원칙
- **기능 차등(게이팅)은 지금, 결제(PG)는 검증 후.** ("수익모델 나중" = 결제가 나중이지 게이팅이 나중이 아님)
- 지표는 **사용자 수보다 사용 강도**: MAU, 주간 재방문율, 1인당 검색 수, 체류시간, 북마크율
  - 예: 50명이 매일 10번 검색 → 유료 가능 / 1000명이 월 1번 → 어려움

## 4. 가격 (3 AI 일치 — 중요)

- ❌ **월 1,000~3,000원은 너무 낮음** → "싸구려" 신호. 폐기.
- ✅ **출시 무료 → 최초 유료 월 4,900원 → 이후 7,900~9,900원**
- 근거: 타깃(회계사·세무사·금융사·법무)은 커피 한 잔(5,000원)으로 30분 절약하면 싸다고 느낌
- 통합: LawQuery 5,900 / AccountingQuery 5,900 / **All Access 9,900** (객단가 상승)

## 5. 결제 구조 (확정)

- **웹에서 PG 정기결제** (토스/포트원). 한국 전기통신사업법으로 외부 PG 가능
- **넷플릭스식**: 앱(TWA)엔 결제·구독 링크 0개(순수 뷰어), 결제는 웹브라우저로 따로
  → 구글 수수료 0%, PCI DSS·24h 보고 API 부담 회피
- 가입(자동승인) ≠ 구독(plan). **plan으로 접근 차등**
- 인프라: 무료 `*.kinphw.kro.kr`(lq./aq.) 유지 → 유료화 검증되면 정식 도메인

## 6. ⚠️ 실패 시나리오 (3 AI 공통 경고 — 회피 필수)

1. **데이터만 모으고 해설 안 붙임** → 국가법령정보센터와 경쟁, 패배. 해설 한 줄이라도 붙여라.
2. **너무 많이 벌림** → 콘텐츠 유지관리 지옥. **초기엔 전자금융 + 회계기준 2개에만 집중.**
3. **유료화를 너무 늦춤** → "1000명 모이면…" 하다 결국 안 붙임. **100명에서 Pro 실험.**

## 7. 현실적 수익 전망

- 월 5,900원 기준: **월 50만 원 ≈ 85명 / 월 100만 원 ≈ 170명** (생각보다 적음)
- 전문직 유료 전환율 10~20% 가능 → 무료 1000명 × 10% = 100명 ≈ 월 59만 원
- **결론: LawQuery+AccountingQuery는 월 50~100만 원 사이드 프로젝트로 충분히 현실적.**
  본업급은 **팀 라이선스(B2B)**가 열쇠.
